진행 단계별 스킬 발동 — v1 vs v2

두 실행의 유일한 차이는 설계 스킬(write-scenario · reference-proposal · ui-design-workflow)과 코딩(create-spec)의 순서다. v1은 코딩을 먼저 해 설계 스킬이 사후 문서화가 됐고 → 접근성 결함을 사후 수정. v2는 설계를 먼저 해 코딩을 이끌어 → 결함을 예방했다.

v1 — 원래 순서 코딩 먼저

  1. boilerplate-setup세팅 · 결정 확정
  2. 디자인 샘플 (Q5*)5안(A/B/C/N/D) → 확정안
  3. create-spec코딩: 5페이지 React 구현
  4. write-scenario이미 만든 화면의 사후 문서화
  5. reference-proposal사후 문서화
  6. ui-design-workflow사후 문서화
  7. ai-usability-testF-001 접근성 결함 발견 (S3=1)
  8. design-handoffHANDOFF.md 생성
  9. run-phase (P1)F-001 수정 — 키보드 접근 12→0

v2 — 정석 순서 설계 먼저

  1. boilerplate-setup세팅 (톤 확정은 뒤로 이연)
  2. write-scenario흐름 · 페르소나 · 과업 (최선행)
  3. reference-proposal기준 톤 확정 (design-direction)
  4. ui-design-workflowflow 락 + "네이티브 a/button" 접근성 요구 명시
  5. create-spec코딩: 설계 이행 · 접근성 선반영
  6. ai-usability-testS3=0 — F-001 예방됨
  7. design-handoffHANDOFF.md 생성
  8. run-phase (P1)검증만 — 수정 불필요
설계 스킬 코딩(create-spec) 사후 문서화 / 재작업 결함 발견 결함 예방 / 검증
